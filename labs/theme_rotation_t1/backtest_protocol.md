# 回测验证协议

本协议用于把 `strategy_spec.md` 落成可复现 Qlib/自研撮合实验。

## 1. 先决条件

开始写回测模块前必须完成：

- `data_requirements.md` 的必需字段审计。
- 所有缺失字段以 `[MISSING]` 写入数据审计报告。
- 规则引擎有 pytest，至少覆盖 T+1、涨跌停不可成交、整手、最低 5 元佣金、事件过滤、setup 停用。
- 不使用公司算力，不使用非公开信息。

## 2. 样本切分

本 lab 不触碰主项目 Phase 4 的正式样本外窗口 `2024-2026.06`，除非用户明确把该 lab 提升为主线候选并冻结规则。

建议 lab 内部先用：

- 训练/规则冻结：2018-2021
- 验证/参数敏感性：2022-2023
- 最终 holdout：`[MISSING]`，在首次运行前写死，且只跑一次

如果数据不足，减少实验范围而不是改规则凑结果。

## 3. 候选生成

候选生成只允许使用买入决策时已经可见的数据：

- 日频 setup 候选默认使用 `t-1` 收盘已知数据。
- 盘中买入门槛使用 `decision_time` 之前的全市场快照。
- 事件风险以公告/披露日生效。
- 题材标签必须有 as-of 生效日期。

## 4. 撮合规则

账户与交易约束：

- 初始资金 `100000`
- 整手 `100` 股
- 总仓位不超过 `85%`
- 现金不低于 `15%`
- T+1
- 涨停不可买，跌停不可卖
- 停牌不可交易

成本：

- 买入佣金：`max成交额 * 0.00025, 5 元`
- 卖出佣金：`max成交额 * 0.00025, 5 元`
- 卖出印花税：`成交额 * 0.0005`
- 股票滑点：`0.15%`

成交价：

- 买入窗口必须落在 10:40 前或 14:40 后。
- 若只有日频数据，不允许验证分时买卖规则，只能做粗粒度候选研究。
- 无分钟数据时，报告标题必须标注“非完整可执行回测”。

## 5. 风控模拟

每个交易日按顺序处理：

1. 更新持仓是否达到 T+1 可卖。
2. 检查止损、止盈、移动止损、分时卖出信号。
3. 处理跌停/停牌导致的未成交。
4. 更新连亏天数、setup 滚动期望、系统滚动期望。
5. 若未停用，再生成新买入候选。
6. 应用市场状态、事件过滤、相关性约束、仓位约束。
7. 生成当日模拟成交和交易建档。

## 6. 统计口径

每笔交易统计：

- 毛收益
- 净收益
- 佣金
- 印花税
- 滑点
- 持有天数
- 最大不利波动
- 最大有利波动
- 是否因涨跌停未成交
- 是否违反或触发冷却规则

setup 级统计：

- 样本数
- 胜率
- 平均盈利
- 平均亏损
- 盈亏比
- 净期望值
- 滚动 30 笔期望值
- 最大回撤
- 摩擦成本占比

系统级统计：

- 年化收益与超额收益
- 最大回撤
- IR
- 年换手
- 年摩擦成本占比
- 滚动 60 笔期望值
- 停用触发次数

## 7. Go/No-Go

不得进入实盘或模拟加仓，除非同时满足：

- 完整可执行回测完成，含分钟数据、T+1、涨跌停不可成交、真实成本。
- 全系统净期望值 `> 0`。
- 每个启用 setup 样本数 `>= 30`，且滚动 30 笔期望值未转负。
- 滚动 60 笔系统期望值未转负。
- 最大回撤未超过预设承受范围。
- 年摩擦成本占比可解释；若超过主项目 `1.5%`，该策略不得并入主线。
- 结果写入 `reports/lab_theme_rotation_t1.md`。

## 8. 报告模板

最终报告至少包含：

- 数据覆盖与 `[MISSING]` 字段
- 样本切分
- 参数版本与 git diff
- 全系统绩效
- setup 分项绩效
- 成本与滑点敏感性
- 涨跌停未成交影响
- 冷却期与停用规则触发记录
- 结论：继续、砍掉、还是回炉

本协议仅供学习研究，不构成投资建议。
