## 项目定位

本阶段产出不是收益额，而是 **① 经过实盘验证的量化系统 ② 12 个月以上可信的真实业绩曲线**。
达不到 Phase 4 准入门槛就不实盘。

| 维度 | 约束 |
|------|------|
| 资金 | 初始 10 万 RMB，`account=100000` 贯穿回测与实盘 |
| 主线 | ETF 板块轮动（60–70% 仓位），月度低频调仓 |
| 辅线 | 3–5 只精选个股（30–40% 仓位），月度调仓 |
| 执行 | 信号自动生成 → **订单清单 + 人工执行 SOP**，禁止自动下单 |
| 考核 | **超额收益与 IR**，不是绝对金额；年摩擦成本 ≤ 1.5% |

## 硬性约束（8 条）

1. **数据不编造**：拿不到的数据标注 `[MISSING]`，禁止用假数据让 pipeline "看起来能跑"
2. **公告日对齐**：财报/公告类因子一律按**公告日**对齐，禁止用报告期日期；上游 altdata 以 `date_published` 生效
3. **真实回测**：所有回测必须以 `account=100000` + **整手（100 股）约束** + `config/costs.yaml` 真实成本模型运行，禁止按理想化百分比权重汇报结果
4. **测试门禁**：每个模块必须有对应的 pytest 测试后才算完成
5. **合规边界**：禁止在代码/注释/文档中引用任何非公开信息源；供应链认知只能来自公开数据或已标注 `[MISSING]` 的字段
6. **人工执行**：本项目不生成任何自动下单代码；执行产出是**订单清单 + 人工执行 SOP**（`sop/`）
7. **密钥安全**：API key 只存 `.env`（已 gitignore），代码/日志/报告中禁止出现明文 key；LLM 调用必须带缓存与用量统计；**不使用公司算力**，LLM 走 OpenAI 兼容 API（`LLM_BASE_URL` / `LLM_MODEL` / `LLM_API_KEY`）
8. **Windows 适配**：文件 IO 显式 `encoding="utf-8"`、环境设 `PYTHONUTF8=1`、路径用 `pathlib`、并行入口加 `if __name__ == "__main__"` 保护、定时任务用**任务计划程序**（非 cron）、不引入仅 Linux 可用的依赖（如 uvloop）

## 小资金三条铁律

所有设计决策的出发点；参数细节见 `config/costs.yaml` / `config/risk.yaml`。

| 铁律 | 关键数字 | 设计后果 |
|------|----------|----------|
| 最低 5 元佣金 | 单笔 < 2 万时 5 元 > 万 2.5 费率 | 个股 ≤ 5 只、单笔尽量 ≥ 6000 元、月度调仓、年换手 ≤ 400% |
| 整手约束 | 一手 = 100 股 | 一手金额 > 12000 元 → `etf_only`，用对应 ETF 表达 |
| 收益期望 | 15% 年化超额 ≈ 1.5 万/年 | 不上杠杆、不做高频、不为"有感觉"而变形 |

**成本模型**（写入 costs.yaml，不得自行发明）：
- 股票：佣金万 2.5 + 最低 5 元 + 卖出印花税 0.05% + 滑点 0.15%
- ETF：佣金万 2.5 + 最低 5 元 + 免印花税 + 滑点 0.05%
- T+1、涨跌停约束必须启用

## 架构约定

```
主线 rotation（指数算信号 → ETF 执行）  65%  │  辅线 stock_select（Top 5）  35%
         ↑                                              ↑
  universe_etf.yaml                          universe_stocks.yaml
  factors/supply_chain.py                    factors/evaluate.py
  models/rotation.py                         models/portfolio.py（整手分配）
```

- **轮动信号在指数上生成**，映射到 `universe_etf.yaml` 对应 ETF 执行
- **供应链传导因子**：Granger 检验滞后 1–6 月，**下游优先用行业指数**（板块 mapping 到单票）
- **因子准入**：|IC| > 0.02 且 ICIR > 0.3 且与已有因子相关 < 0.7 且**月均换手 < 40%**（小资金下一票否决）
- **传导边全部无效则如实记录并砍方向**，不硬凑因子
- **RD-Agent**：可选实验，需 Linux（WSL/云），不阻塞主线，可整体跳过
- **配置即契约**：成本/风控/universe 以 `config/*.yaml` 为准，改参数先改 yaml 再改代码

## 研究与回测规范

| 规则 | 要求 |
|------|------|
| 样本外 | 2018–2023 训练，**2024–2026.06 纯样本外只跑一次**；迭代后该窗口作废须换窗口 |
| 前视偏差 | 因子只能用发布日可见的值；altdata schema：`(indicator_id, date_published, period, value, source_url)` |
| 幸存者偏差 | 回测须包含已退市/被并购标的 |
| 摩擦成本 | 全组合年摩擦 > 1.5% → 强制降频或减少标的数 |
| NLP 全量前 | 必须先跑 100 篇公告成本探针，写入 `reports/phase3_nlp_cost.md` |
| 实盘准入 | 样本外超额 > 0 且 IR > 0.5、最大回撤 < 25%、滑点 0.3% 仍不亏、摩擦 ≤ 1.5% |

**Phase 5 风控硬校验**（`config/risk.yaml`）：单股 ≤ 15%、单 ETF ≤ 30%、科技合计 ≤ 80%、总仓位 ≤ 85%、现金 ≥ 15%

## 红线清单

- ❌ 使用任何非公开信息（供应商内部质量/订单/成本）
- ❌ 回测不带 `account=100000` + 整手 + 最低 5 元佣金就汇报结果
- ❌ 样本外窗口被提前触碰，或看结果后重跑同一窗口
- ❌ 财报因子用报告期而非公告日对齐
- ❌ 为跑通 pipeline 编造数据
- ❌ 上杠杆（融资融券/场外配资）、日内高频化
- ❌ 生成自动下单 / miniQMT / xtquant 代码（升级触发器未到条件前）
- ❌ 手动执行时临场改单且不记入 tracker

## AI 助手工作方式

1. **读 plan.md 当前 Phase**，只做该 Phase 范围内任务，不越级实现
2. **改 config 先读 yaml**，数值以 plan.md 和已有 yaml 为准，plan 未给出的标 `[MISSING]` 或 TODO
3. **供应链是核心 edge**，但验证必须可复现、可检验；认知优势不能替代 Granger/IC 证据
4. **小资金优先**：任何增加换手或摩擦的设计，默认拒绝 unless 有明确量化收益
5. **完成后列出变更清单**，等用户 review；除非明确要求，不自行 commit / push
