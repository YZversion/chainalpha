# 量化研究规划

> **AI工程师视角 × 股票量化 × 供应链传导**
> 你的独特优势：懂AI技术、父亲在联想质量部门 → 你对硬件供应链有比普通散户更深的认知。量化模型的目标是把这种认知优势转化为可验证的交易信号。

---

## 一、你的认知地图：8个领域 × 3条供应链

### 1.1 供应链传导图

```
AI算力需求 ──→ 芯片(GPU/CPU/ASIC) ──→ 显卡/GPU ──→ 电脑设备/数据中心
                     │                      │
                     └─→ PCB(封装基板) ──→ ──┘
                                           │
光纤 ──→ 数据中心 ←── 数据库(数据基础设施)    └─→ 工业设备(制造检测)
```

### 1.2 三条核心传导链

| 传导链 | 路径 | 传导逻辑 | 量化信号 |
|--------|------|---------|---------|
| **算力链** | AI需求 → 芯片 → 显卡 → 数据中心/电脑 | AI训练需求增长 → GPU订单增加 → PCB/封装需求上升 | NVIDIA财报 → 国内GPU公司营收 → PCB龙头订单 |
| **基础设施链** | 数据需求 → 数据库 → 光纤 → 数据中心 | 数据量增长 → 存储需求 → 通信带宽需求 → 服务器部署 | 云厂商资本开支 → 光模块出货量 → IDC租金 |
| **制造链** | 芯片/显卡需求 → 工业设备 → PCB → 电脑设备 | 芯片产量增加 → 检测/制造设备需求 → PCB产能扩张 → 终端出货 | 联想出货量 → PCB龙头产能利用率 → 检测设备订单 |

### 1.3 你父亲的联想质量部门 → 独特信息优势

| 信息维度 | 你能获取的 | 量化价值 |
|----------|-----------|---------|
| 终端出货趋势 | PC/服务器出货量的季度变化 | **终端需求的领先指标**（比财报早1-2个月） |
| 质量问题追踪 | 供应商质量问题的频率和严重性 | **供应商风险的领先信号**（质量问题 → 换供应商 → 订单转移） |
| 供应链切换 | 哪家供应商被引入/被替换 | **订单转移的直接信号**（新供应商→短期业绩弹性） |
| BOM成本变化 | 关键元器件价格走势 | **成本传导的微观验证**（芯片涨价→终端提价能力） |

> ⚠️ **合规提醒**：你父亲分享的信息必须是公开可得或经授权的，不能使用未公开的内幕信息。但"联想PC出货量同比增长10%"这类公开趋势信息是合法的领先指标。

### Phase 2：因子研究 — ⏱ 6-8 周

> 从"定义因子"到"验证因子有没有预测力"。

| 子模块 | 内容 | 产出 |
|--------|------|------|
| **基本面因子** | PE/PB/ROE/营收增速/研发占比的IC检验 | 有效因子列表 |
| **技术面因子** | 动量/反转/波动率/成交量的IC检验 | 有效因子列表 |
| **供应链传导因子** | 上游指标→下游股价的Granger因果检验 | 传导时效和强度 |
| **事件驱动因子** | 政策/财报/产品发布的事件收益率分析 | 事件因子库 |

**供应链传导因子——你最独特的优势**：

| 传导假设 | 上游指标 | 下游标的 | 验证方法 |
|----------|---------|---------|---------|
| AI需求 → GPU公司 | NVIDIA营收增速 | 海光信息/寒武纪股价 | Granger因果 + 滚动回归 |
| GPU需求 → PCB | GPU出货量 | 深南电路/沪电股份 | 领先1-2个月的相关性 |
| 云资本开支 → 光模块 | AWS/阿里云资本开支 | 中际旭创/新易盛 | 领先3-6个月传导 |
| 联想出货 → 供应商 | 联想PC出货量 | 鹏鼎控股/深南电路 | 家人提供的数据辅助验证 |

### Phase 3：模型构建 — ⏱ 8-12 周

> 把有效因子组合成可执行的交易模型。

| 子模块 | 内容 | 产出 |
|--------|------|------|
| **板块轮动模型** | 8个领域间的相对强弱轮动信号 | 月度板块配置建议 |
| **多因子选股模型** | 在每个领域内用多因子筛选个股 | 每板块Top 3个股 |
| **供应链因果模型** | 上游事件 → 下游交易的因果链模型 | 跨板块交易信号 |
| **情绪与动量模型** | 社交媒体情绪 + 技术动量的混合模型 | 短期交易辅助 |

**模型框架选择**：

| 模型 | 框架 | 原因 |
|------|------|------|
| 板块轮动 | 相对强弱RS + 宏观因子 | 简单有效，适合月度调仓 |
| 多因子选股 | IC加权 + 正交化 | 量化经典，可扩展 |
| 供应链因果 | VAR/Granger + 滚动回归 | 你的独特认知优势 |
| 情绪动量 | NLP情绪 + 动量因子 | 你懂AI/NLP，这是强项 |

> 💡 **作为AI工程师，你可以在情绪模型中用LLM做财报/新闻的语义分析**——这是大多数量化散户做不到的。详见 Phase 3 各文件。

> 📌 详见 [Phase 3 → 模型构建](Phase_3_模型构建/01_板块轮动模型.md)

### Phase 4：回测与验证 — ⏱ 4-6 周

> "所有模型在训练数据上看起来都很棒。"——回测是检验真理的唯一标准。

| 子模块 | 内容 | 产出 |
|--------|------|------|
| **回测框架设计** | 滑点/手续费/幸存者偏差的处理规范 | 可信赖的回测引擎 |
| **样本外验证** | 2020-2024训练 → 2025-2026验证 | 真实alpha估计 |
| **模型衰减监控** | 因子IC衰减检测 + 模型定期重训练 | 衰减预警机制 |

**回测铁律**：

| 铁律 | 说明 | 原因 |
|------|------|------|
| **幸存者偏差** | 包含已退市/被并购的公司 | 否则回测收益虚高 |
| **前视偏差** | 因子数据必须用发布时的值 | 否则用了"未来信息" |
| **滑点+手续费** | 每笔交易扣除0.2%+印花税0.05% | 否则回测不可执行 |
| **样本外验证** | 至少2年数据不在训练集中 | 否则只是拟合历史 |
| **滚动窗口** | 因子权重每季度重算 | 否则权重过时 |

> 📌 详见 [Phase 4 → 回测与验证](Phase_4_回测与验证/01_回测框架设计.md)

### Phase 5：实盘与迭代 — ⏱ 持续

> 从回测到实盘，最大的敌人是自己的心态。

| 子模块 | 内容 | 产出 |
|--------|------|------|
| **策略部署** | 模型信号 → 交易执行的自动化流程 | 执行checklist |

**实盘风控参数**：

| 参数 | 建议值 | 说明 |
|------|--------|------|
| 单股最大仓位 | 15% | 量化模型分散风险 |
| 单板块最大仓位 | 30% | 防止行业黑天鹅 |
| 总仓位上限 | 60% | 留40%现金应对意外 |
| 止损线 | 8-10% | 硬止损，无条件执行 |
| 模型信号偏离止损 | 信号连续3次错误 → 降低仓位50% | 模型可能失效 |

## 六、核心原则

1. **数据不编造**：跟Basics文件夹里的原则一致，没查到的标注 [MISSING]
2. **回测不过拟合**：样本外验证是硬要求，训练集表现好不算好
3. **供应链认知是你的alpha**：大部分散户不懂传导链，你懂——这是你的edge
4. **AI能力是第二alpha**：NLP情绪分析、因果推断——散户做不到，你能
5. **风控高于收益**：量化模型亏钱的速度比手动交易更快（机械化执行=机械化亏损）

---

> 💡 **量化不是AI工程师的"降维打击"**——技术能力给了你更好的工具，但市场是另一个维度的游戏。尊重市场的不确定性，用模型管理风险而非追逐收益。

*本规划仅供学习研究参考，不构成投资建议。量化模型存在过拟合、幸存者偏差等多种风险，实盘前务必完成样本外验证。*
