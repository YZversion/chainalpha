# plan.md — A股量化研究系统 v3（10万小资金版）

> 项目代号：`chainalpha`（供应链传导 alpha）
> 环境：WSL2 Ubuntu 22.04 + conda | Python 3.10（Qlib 主环境）/ 3.11（RD-Agent 独立环境）
> 资金约束：初始实盘资金 **10万 RMB**，无券商 API/QMT 权限
>
> **v3 核心定位变化**：本阶段项目的产出不是收益额，而是
> ① 一套经过实盘验证的量化系统 ② 12个月以上可信的真实业绩曲线。
> 资金到位时系统是现成的。因此：**ETF 板块轮动为主线（60-70%仓位）+ 3-5只精选个股为辅线（30-40%仓位），月度低频调仓，信号自动生成、人工执行。**
>
> **给 AI 助手的全局约束（同步进 AGENTS.md / Cursor rules）**：
> 1. 数据不编造：拿不到的数据标注 `[MISSING]`，禁止用假数据让 pipeline "看起来能跑"
> 2. 财报/公告类因子一律按**公告日**对齐，禁止用报告期日期
> 3. 所有回测必须以 `account=100000` + **整手（100股）约束** + 真实成本模型运行，禁止按理想化百分比权重汇报结果
> 4. 每个模块必须有对应的 pytest 测试后才算完成
> 5. 禁止在代码/注释/文档中引用任何非公开信息源
> 6. 本项目不生成任何自动下单代码；执行环节的产出是**订单清单 + 人工执行 SOP**

## 小资金三条铁律（所有设计决策的出发点）

| 铁律 | 数字 | 设计后果 |
|------|------|---------|
| **最低5元佣金** | 单笔 < 2万元时，5元佣金 > 万2.5费率；3300元/笔时成本达0.15%/边 | 个股持仓数 ≤ 5只、单笔金额尽量 ≥ 6000元、月度调仓、目标年换手 ≤ 400% |
| **整手约束** | 一手=100股；高价股一手数万元 | universe 剔除"一手金额 > 12000元"的个股（约仓位40%的1/3），高价标的用对应ETF表达 |
| **收益期望** | 15%年化超额 ≈ 1.5万/年 | 不上杠杆、不做高频、不为"收益有感觉"而变形；考核指标是**超额收益与IR**，不是绝对金额 |

### 3.2 个股精选（辅线，第3-5周）

- [ ] `workflows/lgb_stock_select.yaml`：Alpha158 + 自有因子，LightGBM，**月度调仓，Top 5**（不是Top 30）

### 3.3 NLP 情绪模块（第5-7周）

## Phase 5：实盘（人工执行版）— ⏱ 持续

> 本 Phase 无自动下单。月度 10-16 笔委托，人工执行 10 分钟，自动化在此规模下是伪需求且引入风险。

### 5.1 执行系统（第1-2周）

- [ ] `execution/order_sheet.py`：调仓日自动生成订单清单——`(标的, 方向, 股数[整手], 限价建议, 预估金额, 风控检查结果)`，输出 markdown/表格，推送到手机（如 server酱/邮件）
- [ ] `execution/risk_check.py`：订单清单生成前硬校验——单股≤15%、单ETF≤30%、科技合计≤80%（注：本项目就是科技主题，此约束防单一子板块过度集中，另建议**账户外**保留家庭应急资金）、总仓位≤85%、现金≥15%
- [ ] `sop/monthly_rebalance.md`：人工执行 SOP——执行时间（建议开盘后30分钟避开集合竞价波动）、限价单规则、部分成交处理、当日未成交次日处理
- [ ] `sop/stop_loss.md`：止损 SOP——券商 APP 条件单设置方法（个股-10%/ETF-12%），每月调仓日核对条件单是否齐全
- [ ] `execution/tracker.py`：每笔实盘成交录入（手动CSV或对账单导入）→ 自动计算实际滑点、实际佣金、实盘vs回测日度偏差

### 5.2 上线（第3周起）

- [ ] 模拟盘先行：聚宽/果仁建同策略模拟组合做**公开存证**，与实盘并行
- [ ] 实盘第1-3个月：每次执行后当天填 tracker；月末生成偏差报告
- [ ] 偏差验收：实盘月收益与回测偏差的主要来源分解（滑点/未成交/整手误差），连续3个月偏差可解释且滑点实际值 ≤ 回测假设 → 系统判定"可信"

### 5.3 迭代与升级路径

- [ ] 月度复盘模板：alpha衰减？传导逻辑变化？摩擦成本实际值？
- [ ] 季度：滚动重训 + 因子库审查
- [ ] **升级触发器（写死，到条件才做）**：
  - 资金 ≥ 30万 或 实盘曲线验证12个月有效 → 询价 2-3 家券商 miniQMT（门槛可谈），进入半自动（自动生成订单+人工确认发送）
  - 资金 ≥ 50万 且 半自动稳定6个月 → 启用完整自动化（另立 plan v4，含 xtquant 封装、dry_run 灰度）

**DoD**：实盘6个月零执行事故（无漏止损单、无超风控订单）；tracker 数据完整；模拟盘存证同步运行。

## 红线清单

- ❌ 使用任何非公开信息（供应商内部质量/订单/成本信息）
- ❌ 回测不带 `account=100000` + 整手 + 最低5元佣金就汇报结果
- ❌ 样本外窗口被提前触碰，或看结果后重跑同一窗口
- ❌ 财报因子用报告期而非公告日对齐
- ❌ 为跑通 pipeline 编造数据
- ❌ 上杠杆（融资融券/场外配资）、日内高频化
- ❌ 手动执行时临场改单（偏离订单清单需在 tracker 记录原因，连续发生 → 复盘心态问题）
